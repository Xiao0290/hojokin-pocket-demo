## Status

`v0.1-local-mvp` is the frozen local demo baseline for Closed Beta development. It is suitable for professional validation meetings and internal product testing, but it is not production-ready.

The attached Closed Beta plan referenced an older main commit. This release note intentionally records the current real baseline after the responsive customer demo, diagnosis provenance, and solution-first flow PRs were merged.

## Included Capabilities

- Responsive customer demo shell across desktop, tablet, and mobile sizes.
- URL diagnosis using extracted company profile signals.
- Matching results that expose provenance and avoid a fixed universal recommendation set.
- Subsidy detail view with requirements, source metadata, and applicant confirmations.
- Administrative-scrivener consultation CTA as the primary customer solution path.
- Applicant-owned business-plan draft preparation after the consultation path.
- DOCX/PDF export with AI draft disclaimer preservation.
- Expert waitlist and notification/analytics proof points for local validation.
- Local source-review/admin proof of concept for future reviewed-data workflows.
- Issue / PR / CI / handoff workflow documentation and readiness scoring.

## Known Limits

- Persistence is local JSON-backed MVP storage.
- Auth is local/mock only and not production authentication.
- Tenant isolation is implemented as a local access boundary, not production multi-tenant auth.
- Subsidy data is fixture/source-backed MVP data, not official full-coverage ingestion.
- Source review exists as a local proof of concept; production reviewed-data publishing is not complete.
- Draft generation is deterministic template logic with `llm: false`, not Bedrock/RAG.
- No official filing, proxy application, jGrants POST, or gBizID write-side integration exists.
- No paid expert marketplace, success-fee, platform take-rate, or adoption-probability claim is enabled.
- Closed Beta legal terms, data license decisions, backup/restore, observability, invite gating, and production deployment are future work.
